# Persona Writing

Ask the user to confirm copy style before writing personas.

Suggested style choices:

- warm and emotional;
- professional assessment;
- young social product;
- brand marketing;
- concise operational.

## Persona Requirements

Each persona must include:

- deterministic key;
- human-readable name;
- one-sentence description;
- full report;
- suitable counterpart or fit guidance;
- optional image prompt.

Use positive framing by default. Avoid clinical diagnosis language unless explicitly requested.

## Consistency Rules

- Persona names should be distinct and memorable.
- Persona descriptions should reflect the exact axis combination.
- Reports should cover behavior, needs, likely relationship pattern, and suitable counterpart.
- Similar adjacent personas should share family resemblance but differ clearly on the changed axis.

## Image Prompt Rules

When images are requested:

- generate one consistent art direction first;
- write one prompt per persona;
- include pose, mood, palette, background, and symbolic elements;
- avoid hardcoding a model name;
- use the current available image generation capability.
