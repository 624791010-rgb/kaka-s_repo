# File Responsibilities

## Stable entry files

### `ai/prd.md`
- Points to the real product requirement sources
- Should not duplicate the full PRD unless the repo truly has no other source

### `ai/global-rules.md`
- Long-lived engineering rules
- Stack, workflow constraints, shared contracts, high-risk files, reading order

## Reference files

### `ai/reference/git-workflow.md`
- Git safety, snapshot habits, staging rules, dirty worktree handling

### `ai/reference/project-map.md`
- High-level repo map: apps, backend, docs, SQL, shared entrypoints

### `ai/reference/qa-checklist.md`
- Stable smoke tests and regression checklist

### `ai/reference/style-guide.md`
- Visual and writing style only if UI work is frequent

### `ai/reference/component-guide.md`
- Reuse and composition rules for shared UI pieces

### `ai/reference/api-conventions.md`
- Client/server interface norms, response shapes, naming rules

### `ai/reference/piv-roadmap.md`
- Current PIV ordering and boundaries

### `ai/reference/handoff-workflow.md`
- What to do when the user asks for handoff/closure

### `ai/reference/handoff-template.md`
- Fixed template for each handoff doc

## Command files

### `ai/commands/prime.md`
- The startup sequence for a new window

### `ai/commands/plan-feature.md`
- Optional feature planning checklist for larger work

### `ai/commands/commit.md`
- Optional commit preparation checklist

## Archive files

### `ai/archive/handoff_*.md`
- Iteration-specific handoff packages

### `ai/archive/latest.md`
- The single stable pointer to the newest handoff package
